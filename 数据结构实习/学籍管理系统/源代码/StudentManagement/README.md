学籍管理系统 v2.0 - 数据结构大作业

项目简介

这是我做的数据结构课程大作业，一个用C++写的学籍管理系统。简单说就是一个能管理学生、课程、选课和成绩的小软件，有点像简化版的教务系统。

程序是按照多文件的方式写的，不同的功能放在不同的文件里，这样结构比较清楚，改代码的时候也好找。系统里分两种用户：管理员（老师）和学生，他们能做的事情不一样，比如学生只能看自己的信息，管理员什么都能管。

为了防止乱输数据，我还加了很多检查。比如学号必须8位数字，生日要输对（连闰年都能判断），成绩必须是0到100分等等。输错了程序会直接告诉你，让你重新输。

1. 学生信息 - 顺序表 (vector)
2. 课程信息 - 顺序表 (vector)  
3. 选课记录 - 单链表
4. 成绩记录 - 单链表
5. 快速查询 - 二叉排序树 (BST)

主要功能

1. 学生管理
添加新生，要填学号、姓名、性别、生日、联系方式这些
删除学生的话，他选过的课和成绩也会一起删掉，不会留垃圾数据
能按学号或者名字找人
可以把所有学生列出来看

2. 课程管理
能添加新开的课，要填课号、课名、上课时间地点、学分这些
能看现在开了哪些课

3. 选课管理
学生可以自己选课，选完会记下来
也能退课
能查某个学生选了哪些课

4. 成绩管理
老师可以给学生录成绩
学生只能看自己的成绩，还会自动算个平均分
成绩可以改

5. 其他功能
能综合查询，按不同条件找人
有些统计功能，比如看看现在系统里有多少学生、多少门课
用户管理，管理员可以给学生开账号

技术实现

用了哪些数据结构

这个项目重点就是展示怎么用不同的数据结构解决不同的问题。

1. vector（顺序表）
  用来存学生和课程的完整信息
  好处是遍历起来快，要显示所有学生的时候特别方便
  实现简单，C++自带的

2. 链表
   用来存选课记录和成绩记录
   因为选课退课很频繁，链表插入删除特别快
   虽然查找某个特定记录有点慢，但通常我们是要看一个学生的所有选课，这正好需要遍历，链表挺合适

3. 二叉排序树（BST）
   用来给学生建索引，实现快速查找
   单纯用vector找人的话，要从头到尾一个个比，太慢了
   用二叉排序树，可以做到大概log(n)的时间找到人，快很多
   而且我用了AVL树（一种平衡二叉搜索树），因为学号经常是连续的，普通BST会变成一根“长棍子”，查找又慢了。AVL树能自己保持平衡，确保查找一直很快

怎么让数据不丢

所有数据都会自动保存到data文件夹的文本文件里。包括：
users.txt - 存用户账号密码
students.txt - 存学生信息
courses.txt - 存课程信息
selections.txt - 存谁选了哪门课
scores.txt - 存成绩

程序一打开就会从这些文件读数据，退出或者改了数据会自动保存回去，这样下次打开数据还在。

怎么用

登录

有两个现成的账号可以用：
管理员：admin / admin123 （什么都能管）
学生：stu001 / 123456 （学号20240001，只能看自己的东西）

管理员能做什么

登录之后会看到这些选项：
1. 管学生 - 增删改查学生信息
2. 管课程 - 增删改查课程
3. 管选课 - 看谁选了啥课
4. 管成绩 - 录成绩、改成绩
5. 查询统计 - 各种查询和统计
6. 管用户 - 给学生开账号
0. 退出

学生能做什么

学生登录后只能：
1. 看自己的信息
2. 选课/退课
3. 看自己选了哪些课
4. 看自己的成绩和平均分
5. 改密码
0. 退出

输入要注意的

系统会检查你输的对不对：
学号：8位数字，比如20240001
课号：2-3个大写字母+3数字，比如CS101
电话：11位，1开头
生日：2000-01-01这种格式，闰年也能判断
性别：只能是男或女
成绩：0-100分
学分：1-8分
用户名：3-20位字母数字下划线
密码：6-20位
输错了会提示，要重新输。

设计细节

类怎么组织的

基本上一个功能一个类：
User、UserManager - 管用户登录
Student、StudentManager - 管学生，用了AVL树来快速查找
Course、CourseManager - 管课程
SelectionManager - 管选课，用链表
ScoreManager - 管成绩，也用链表
System - 主程序，协调上面这些
Utils - 一些工具函数，比如检查数据对不对

数据存储

存在文本文件里，用|分隔不同字段，比如：

20240001|张三|男|2000-01-01|团员|13800138000|北京市


性能怎么样

按学号找学生：O(log n)，用AVL树很快
添加学生：O(log n)，主要花在更新AVL树上
删除学生：O(n) + O(log n)，从vector删是O(n)，从AVL树删是O(log n)
选课/录成绩：O(1)，链表头插很快
看所有学生：O(n)，vector遍历一遍

注意事项

1. 管理员账号admin不能删
2. 删学生时会把他选的课和成绩一起删了
3. 不能重复选同一门课
4. 只能给已经选了课的学生录成绩
5. 第一次运行要确保有data文件夹
6. 中文显示正常，用了UTF-8编码
